Methods: .createElement and .assign

( Create an interReaction Custom Element and Assign a <%= htmlWebpackPlugin.options.buildName %> component instance to an interReaction Custom Element )

The .createElement method creates and returns a new interReaction Custom Element. You may then add this element to the DOM, as you would any HTML element.

The .assign method assigns <%= htmlWebpackPlugin.options.buildName %> component instances to interReaction Custom Element nodes - usually, Custom Elements created using interReaction's .createElement method, then added to the DOM (such as with Element.appendChild, Element.insertBefore, etc.). Also, accepts a props object, the members of which, as the name implies, are passed as props, to the <%= htmlWebpackPlugin.options.buildName %> component instance.

Method: .createAssignToDomElement

( Create an interReaction Custom Element, append it to the DOM, and assign its <%= htmlWebpackPlugin.options.buildName %> component instance, in one method )

The .createAssignToDomElement method combines the functionality of methods .createElement and .assign. It creates and adds the interReaction Custom Element to the DOM, according to the DOM node provided. Additionally, it does so with positioning, relative to the DOM node's children, according to an additional arg of 'append' (which adds it to the tail of the node), 'prepend' (which adds it to the head of the node), or 'replace' (which replaces all children of the node). The default (no-arg) is 'append'.

As with the .assign method, it can also take props, to be passed to the <%= htmlWebpackPlugin.options.buildName %> component instance.

It is a most convenient, most succinct means of adding one or more <%= htmlWebpackPlugin.options.buildName %> component instances, but lacks the flexibility of using the methods .createElement and .assign separately. That said, it is still very useful, and may even be your regular go-to method, for adding <%= htmlWebpackPlugin.options.buildName %> components.
